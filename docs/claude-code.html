Claude Code Starter Rules 💻

From @techyai_cj. Claude Code reads a file called CLAUDE.md in your project and follows whatever rules you put there, every session, forever. These are the starter rules I recommend to everyone. Copy, paste, edit the bracketed parts.

How to set it up (60 seconds)

1. Open your project folder in Claude Code.

2. Create a file named CLAUDE.md in the project root (or run /init and let Claude draft one).

3. Paste the rules below and edit them to your project. Done. It reads them automatically from now on.

The starter rules

# Project rules

## How to work with me
- Before changing anything, give me a short plan and wait for my ok.
- Explain what you changed in plain English, one short paragraph.
- If you are not sure what I meant, ask. Do not guess on big decisions.
- When something fails, show me the actual error before trying fixes.

## Code style
- Match the style that already exists in this project.
- Small changes over big rewrites. Never refactor what I did not ask about.
- No new dependencies without asking me first.

## Off limits
- Never touch: [folders or files you want protected, like .env, secrets, deploy scripts]
- Never commit or push unless I say so.

## About this project
- What it is: [one sentence about your project]
- How to run it: [the command, like npm run dev]
- How to test it: [the command, like npm test]

Why these work: the plan-first rule saves you from surprise rewrites, the plain English rule keeps you in control even if you do not read every line, and the off-limits list is your seatbelt. The project section pays off forever: Claude stops asking you how to run things.

Level up: you can also say "add a rule to CLAUDE.md: [rule]" whenever Claude does something you never want again. The file grows with you. That is the whole trick.
